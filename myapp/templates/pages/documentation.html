Installation

info

Note

LatexGenius is primarily a web-based platform and doesn't require local installation. This section is for self-hosting the open-source version.

Prerequisites

  • check_circle Python 3.10+
  • check_circle Django 5.2+
  • check_circle Docker (for LaTeX compilation service)

Quick Setup

# Clone the repository
git clone https://github.com/anomalyco/latexgenius.git
cd latexgenius

# Create virtual environment
python -m venv venv

# Activate on Windows
venv\Scripts\activate

# Or activate on macOS/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

Environment Configuration

# Copy the example env file
cp .env.example .env

# Edit .env with your configuration
# Required variables:
# - DATABASE_URL: SQLite or PostgreSQL connection string
# - LATEX_COMPILER_URL: LaTeX.Online service URL
# - SECRET_KEY: Django secret key

# Start the LaTeX compiler service
docker-compose up -d latex-compiler

# Run migrations
python manage.py migrate

# Create admin user
python manage.py createsuperuser

# Start development server
python manage.py runserver

Template Structure

All templates follow a consistent structure that the AI uses to map your content:

% ============================================
% Document Class & Packages
% ============================================
\documentclass[journal]{IEEEtran}
\usepackage{amsmath,amssymb}
\usepackage{graphicx}
\usepackage{cite}

% ============================================
% Title & Authors
% ============================================
\title{Your Paper Title}
\author{Author One, Author Two}

% ============================================
% Abstract & Keywords
% ============================================
\begin{abstract}
Your abstract content goes here...
\end{abstract}

\begin{IEEEkeywords}
keyword1, keyword2, keyword3
\end{IEEEkeywords}

% ============================================
% Main Content
% ============================================
\begin{document}
\maketitle

\section{Introduction}
% Your introduction...

\section{Methodology}
% Your methodology...

\section{Results}
% Your results...

\section{Conclusion}
% Your conclusion...

% ============================================
% References
% ============================================
\bibliographystyle{IEEEtran}
\bibliography{references}

\end{document}

Troubleshooting

Common issues and their solutions.

Common causes and fixes:

  1. Check for unmatched braces { or }
  2. Ensure all \usepackage{} declarations are in the preamble
  3. Verify image filenames don't contain spaces or special characters
  4. Check the compilation log for specific error messages

Tips for better AI conversion:

  • Use clear section headings (Introduction, Methodology, etc.)
  • Place equations on separate lines
  • Include references in a dedicated section at the end
  • Avoid complex nested tables in source documents
  • Try converting from Markdown or plain text instead of Word for more predictable results

Image troubleshooting:

  • Use PDF, PNG, or JPEG formats (EPS is deprecated)
  • Ensure filenames have no spaces or special characters
  • Check that the graphicx package is included
  • Use relative paths without leading slashes: \includegraphics{images/fig.pdf}

BibTeX troubleshooting checklist:

  1. Ensure you have \bibliographystyle{...} and \bibliography{...} in your document
  2. Verify that all citation keys in \cite{} match entries in your .bib file
  3. Check that your .bib file has proper syntax (comma-separated entries, matching braces)
  4. The compilation requires multiple passes: LaTeX → BibTeX → LaTeX → LaTeX
